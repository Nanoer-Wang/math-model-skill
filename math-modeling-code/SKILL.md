# 数学建模代码 Skill

## 目标
把建模方案变成可运行、可复现、可维护的代码。代码应服务论文结果，不追求过度工程化，但必须结构清楚、参数可改、结果可复现。

## 触发场景
- 用户要求写 Python/MATLAB 代码实现模型。
- 用户要求调试建模代码、优化求解代码、机器学习训练代码。
- 用户需要从数据到结果的一键运行脚本。
- 用户需要保存预测结果、评价指标、消融实验、敏感性分析。

## 不应触发
- 只做模型路线讨论时，转交建模 Skill。
- 只做绘图美化时，转交绘图 Skill。
- 只做论文段落时，转交写作 Skill。

## 输入
尽量收集：
- 数据文件路径、字段含义、目标变量。
- 建模公式或算法说明。
- 运行环境：Python/MATLAB 版本、依赖、操作系统。
- 输出要求：表格、图片、指标、预测文件、报告。

## 工作流
1. 检查项目结构：确认 data、scripts、src、outputs、figures、reports 是否存在。
2. 读取数据：写健壮的数据加载逻辑，检查文件存在、字段完整、编码和单位。
3. 数据预处理：缺失值、异常值、标准化、特征工程、训练测试划分。
4. 模型实现：先写稳妥 baseline，再写增强模型或优化算法。
5. 评价指标：根据任务输出 RMSE、MAE、MAPE、R2、Accuracy、F1、AUC、目标函数值等。
6. 实验记录：保存参数、随机种子、指标、预测结果、中间数据。
7. 可复现入口：提供 `main.py` 或清晰脚本顺序。
8. 错误处理：对缺文件、字段名不匹配、样本不足、求解失败给出清楚报错。

## Python 代码规范
- 使用 `pathlib.Path` 管理路径。
- 固定随机种子。
- 函数名表达用途，避免所有逻辑堆在一个脚本里。
- 输出文件统一保存到 `outputs/`、`figures/`、`tables/`。
- 对关键参数集中配置，例如 `config.yaml` 或脚本顶部常量。
- 大型模型应保存训练日志和指标 JSON。

## 推荐项目结构
```text
src/
  data.py
  features.py
  models.py
  metrics.py
  optimize.py
  utils.py
scripts/
  01_collect_data.py
  02_clean_data.py
  03_train_baseline.py
  04_train_final_model.py
  05_sensitivity_analysis.py
outputs/
  metrics.json
  predictions.csv
  model_summary.md
figures/
tables/
```

## 输出格式
默认输出：
- 可运行代码。
- 依赖安装说明。
- 运行命令。
- 输入输出文件说明。
- 指标结果。
- 潜在问题和后续改进。

## 质量标准
- 代码必须能从原始/处理后数据生成论文所需结果。
- 不允许把关键结果硬编码进代码。
- 每个随机过程必须设置随机种子。
- 每个结果表或图都应能追溯到脚本。
- 算法失败时必须返回可诊断信息。

## 禁止事项
- 不要写不可运行的伪代码冒充完整代码。
- 不要无依据地生成随机数据代替真实数据。
- 不要在代码中静默吞掉异常。
- 不要把所有图、表、模型结果混在一个难维护脚本里。
