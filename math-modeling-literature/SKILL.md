# 数学建模文献 Skill

## 目标
为数学建模比赛或课题快速建立可信背景、方法谱系、评价指标和创新切入点。输出应服务于后续数据、建模、写作和排版流程。

## 触发场景
- 用户要求查找、整理、总结某个建模题目的背景资料。
- 用户需要找已有模型、算法、评价指标、政策背景或行业报告。
- 用户要求为论文写“问题背景”“国内外研究现状”“方法综述”“参考文献”。
- 用户想判断某个模型或创新点是否已有类似工作。

## 不应触发
- 只需要写代码、画图或排版时，不要单独使用本 Skill。
- 用户明确已有完整文献并只要求润色时，转交写作 Skill。

## 输入
尽量收集以下信息：
- 题目原文、赛题附件、研究对象、时间范围。
- 用户指定的关键词、学科方向、地区或行业。
- 是否需要中文文献、英文文献、政策文件、统计年鉴、行业报告。
- 输出用途：建模假设、背景介绍、方法选择、论文引用或创新点设计。

## 工作流
1. 解析题目：识别问题类型，例如预测、评价、优化、机理建模、仿真、调度、网络传播、资源配置。
2. 构造关键词：中文关键词、英文关键词、同义词、专业术语、数据源关键词。
3. 分层检索：优先学术论文、官方统计/政策、权威报告，再考虑新闻和普通网页。
4. 提炼已有方法：记录每类方法的适用条件、优缺点、常用变量、评价指标和可复现难度。
5. 形成建模建议：给出 2–4 条可落地技术路线，标注难度、数据需求、创新空间。
6. 产出引用素材：保存题名、作者、年份、来源、DOI/URL、核心结论和可引用位置。

## 输出格式
默认输出：
1. 题目背景摘要。
2. 关键概念解释。
3. 相关研究/方法分类表。
4. 可用数据源清单。
5. 推荐建模路线。
6. 可写进论文的研究现状段落。
7. 参考文献条目草稿。

## 质量标准
- 外部事实必须有来源，不能编造参考文献。
- 区分“权威事实”“文献观点”“建模假设”“个人建议”。
- 文献数量不必过多，但必须服务建模。
- 优先找能支撑变量选择、模型选择、参数范围和评价指标的资料。
- 给后续 Skill 留下明确交接信息：关键词、数据源、可用模型、引用条目。

## 禁止事项
- 不要为了显得高级而堆砌无关论文。
- 不要直接复制大段文献原文。
- 不要把未经验证的网页内容当成权威结论。
- 不要只给泛泛的“可以用机器学习/优化算法”，必须说明为什么适合本题。
