# 数学建模排版 Skill

## 目标
将建模论文、图表、公式、参考文献和附录整理成符合比赛或课程要求的最终提交文件。重点是格式统一、编号正确、交叉引用完整、提交材料齐全。

## 触发场景
- 用户要求排版数学建模论文。
- 用户要求整理 LaTeX、Word、Markdown 到 PDF。
- 用户要求修复公式、图表、目录、参考文献、附录格式。
- 用户要求按照 CUMCM、MCM/ICM 或学校模板排版。
- 用户要求最终提交前检查。

## 不应触发
- 只写论文内容时，转交写作 Skill。
- 只画图时，转交绘图 Skill。
- 只跑代码时，转交代码 Skill。

## 输入
尽量收集：
- 比赛/课程模板文件。
- 论文正文、图片、表格、参考文献、附录代码。
- 格式要求：页数、字体、字号、行距、页边距、编号规则、匿名要求。
- 输出格式：PDF、docx、tex、zip。

## 工作流
1. 检查模板：确认封面、摘要、正文、参考文献、附录结构。
2. 统一样式：标题层级、正文字体、行距、页边距、段前段后。
3. 公式处理：使用规范数学环境，公式编号连续，变量斜体/单位正体。
4. 图表处理：图题表题、编号、交叉引用、分辨率、位置。
5. 参考文献：统一 GB/T 7714、APA、IEEE 或比赛要求格式。
6. 附录组织：代码、额外结果、数据说明不要挤占正文逻辑。
7. 编译/导出：保证无报错、无缺图、无乱码、无空引用。
8. 提交检查：文件命名、匿名要求、页数限制、附件完整性。

## LaTeX 建议
- 使用 `latexmk` 或 `xelatex` 编译中文论文。
- 中文论文建议 `ctex` 系列模板。
- 图片统一放在 `figures/`，表格可放在 `tables/` 或正文。
- BibTeX/Biber 参考文献文件放在 `refs.bib`。
- 交叉引用使用 `\label{}` 与 `\ref{}`，不要手写“图1”。

## Word 建议
- 使用样式管理标题，不要手动调每个标题。
- 图题表题使用题注功能。
- 目录和编号使用自动更新。
- 公式尽量用内置公式编辑器或 MathType，保持统一。

## 输出格式
默认输出：
- 排版后的主文件。
- PDF 导出文件。
- 图片/表格/参考文献检查报告。
- 最终提交清单。

## 质量标准
- 图表和公式编号必须连续、引用正确。
- 参考文献在正文中有引用，正文引用在文末有条目。
- 不能出现“??”“Citation needed”“图见下方”等未完成标记。
- 图片不能模糊，表格不能跨页混乱。
- 提交文件命名和匿名要求必须符合赛题。

## 禁止事项
- 不要为了排版删除关键信息。
- 不要手工硬改编号导致交叉引用失效。
- 不要用低清截图替代可矢量化图。
- 不要忽略比赛模板中的特殊要求。
